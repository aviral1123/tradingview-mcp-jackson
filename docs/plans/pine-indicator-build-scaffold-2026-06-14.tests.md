## 2. Unknown indicator name → error, non-zero exit (unit)

- `node scripts/pine_build.mjs nonexistent-indicator` → exits with code 1 (or non-zero); stderr contains the indicator name and something like "not found" or "no such indicator"

## 3. Path traversal rejection (unit/security)

- `node scripts/pine_build.mjs ../../../etc/passwd` → non-zero exit; must NOT attempt to read that path; error message identifies the illegal traversal
- `node scripts/pine_build.mjs foo/../../../secret` → same
- Assertion: the resolved path must start with `<repo-root>/indicators/` — any resolution outside that prefix is rejected before any file I/O

## 4. Source lint — bias-stack/source.pine (static, no TV)

Assert all of the following for `indicators/bias-stack/source.pine`:
- First non-comment, non-blank line is `//@version=6`
- File contains `indicator(` declaration with `overlay=true`
- File contains at least one `input.int(` or `input.float(` call (adjustable inputs present)
- File contains `emaFast` (or equivalent) with default value `20` visible in an `input.*` call
- File contains `emaSlow` (or equivalent) with default value `50`
- File contains `rsiLen` (or equivalent) with default value `14`
- File contains `ta.crossover(` (the RSI 50 trigger)
- File contains `bgcolor(` (the bias shading)
- File contains `plotshape(` (the triangle marker)
- File does NOT contain `strategy.entry` or `strategy.close` (advisory-only)
- File does NOT contain `strategy(` as the main declaration (must be `indicator(`)
